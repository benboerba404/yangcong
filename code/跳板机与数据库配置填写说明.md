# 跳板机与数据库配置填写说明

使用前请先：

1. 将 **`jump_export_config.example.json`** 复制并重命名为 **`jump_export_config.json`**（该文件已被 .gitignore 忽略，不会提交）。
2. 在 **`jump_export_config.json`** 中，把下面每一项的占位符替换成你的真实信息。

---

## 一、跳板机（SSH）

| 配置项 | 说明 | 填写示例 |
|--------|------|----------|
| **ssh_host** | 跳板机 IP 或域名 | `221.194.189.145` |
| **ssh_user** | 跳板机 SSH 登录用户名 | `master` |
| **ssh_password** | 跳板机 SSH 登录密码 | （你的密码） |

---

## 二、数据库

| 配置项 | 说明 | 填写示例 |
|--------|------|----------|
| **db_host** | 数据库在内网的 IP（跳板机可访问） | `10.17.2.45` |
| **db_port** | 数据库端口（10010 常用为 Spark SQL） | `10010` |
| **username** | 数据库用户名 | `tanchen` |
| **password** | 数据库密码 | （你的密码） |
| **database** | 默认库名（执行 SQL 时的默认库） | `tmp` 或 `crm` |

---

## 三、依赖安装（一次性）

在命令行执行：

```bash
pip install sshtunnel impyla pandas openpyxl
```

| 包名 | 作用 |
|------|------|
| sshtunnel | 建立 SSH 隧道连接跳板机 |
| impyla | 连接 Impala/Spark SQL |
| pandas | 查询结果转成表格 |
| openpyxl | 导出为 .xlsx |

若你之前一直用旧版 `impala` 包，可改为：`pip install impala`，脚本中的连接方式仍可用。

---

## 四、填写完成后

- 确认 **`jump_export_config.json`** 中已无「请填写」等占位符，且密码为真实值。
- 运行导出脚本时加上：`--config code/jump_export_config.json`（若脚本默认就读该路径则可不加）。
